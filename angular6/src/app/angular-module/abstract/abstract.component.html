JavaScript 和 Angular 都使用模块来组织代码，虽然它们的组织形式不同，但 Angular 的应用会同时依赖两者

JavaScript 模块

模块是内含 JavaScript 代码的独立文件

使用：export && import 用来防止全局变量引起的意外

NgModules

NgModule 是一些带有 @NgModule 装饰器的类

imports 数组中的这些模块与 JavaScript 模块不同，它们都是 NgModule 而不是常规的 JavaScript 模块

NgModule和JavaScript模块不同点
  • Angular 模块只绑定了可声明的类，这些可声明的类只是供Angular 编译器用的
  • 与 JavaScript 类把它所有的成员类都放在一个巨型文件中不同，你要把该模块的类列在它的 @NgModule.declarations 列表中
  • Angular 模块只能导出可声明的类。这可能是它自己拥有的也可能是从其它模块中导入的。它不会声明或导出任何其它类型的类
  • 与 JavaScript 模块不同，NgModule 可以通过把服务提供商加到 @NgModule.providers 列表中，来用服务扩展整个应用

基本结构

@NgModule 装饰器表明 AppModule 是一个 NgModule 类

  • declarations —— 该应用所拥有的组件
  • imports —— 导入 BrowserModule 以获取浏览器特有的服务，比如 DOM 渲染、无害化处理和位置（location）
  • providers —— 各种服务提供商
  • bootstrap —— 根组件，Angular 创建它并插入 index.html 宿主页面
declarations 数组
  • 每个组件都应该（且只能）声明（declare）在一个 NgModule 类

  • declarations 数组只能接受可声明对象

    可声明对象包括组件、指令和管道

  • 一个模块的所有可声明对象都必须放在 declarations 数组中
使用 declarations 数组声明指令
  • 从编写它的文件中导出它
  • 导入到适当的模块中
  • 在 @NgModule 的 declarations 数组中声明
imports 数组

模块的 imports 数组只会出现在 @NgModule 元数据对象中

组件的模板中可以引用在当前模块中声明的或从其它模块中导入的组件、指令、管道

bootstrap 数组

通过引导根模块启动

Angular 模块把组件、指令和管道打包成内聚的功能块，每个模块聚焦于一个特性区域、业务领域、工作流或通用工具

NgModule的元数据
  • 声明某些组件、指令和管道属于这个模块
  • 公开其中的部分组件、指令和管道，以便其它模块中的组件模板中可以使用它们
  • 导入其它带有组件、指令和管道的模块，这些模块中的元件都是本模块所需的
  • 提供一些供应用中的其它组件使用的服务

自带模块

NgModule 引用路径 使用
BrowserModule @angular/platform-browser 在浏览器中运行应用
CommonModule @angular/common 使用NgIf或NgFor
FormModule @angular/forms 模板驱动表单（包含 NgModel ）
ReactiveFormModule @angular/forms 响应式表单
RouterModule @angular/router 要使用路由功能，RouterLink,.forRoot()、 .forChild() 时
HttpClientModule @angular/common/http 和服务器对话时
注意：BrowserModule模块最先被导入在跟模块中
